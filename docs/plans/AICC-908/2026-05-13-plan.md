## 1. 티켓 원문

### 배경/목적
도메인 모델(`CallbotAgent`, `CallbotMembership`)과 `transfer_to_agent` 도구는 존재하나, 실제 통화 중 `branch_trigger` 조건을 평가해 sub 봇으로 컨텍스트를 이관하는 런타임 로직이 없다. `voice_session.py` 에서 `transfer_to_agent` 호출 시 대상 봇 세션으로 실제 전환이 일어나야 한다.

### 작업 내용
- `voice_session.py`: `transfer_to_agent` 호출 시 현재 세션 컨텍스트(대화 이력 + 변수)를 sub 봇에 전달하고 세션 교체
- `branch_trigger` 문자열 평가 로직 구현 (LLM 판단 or 키워드 매칭)
- 인계 시 greeting 없이 바로 이어서 대화 시작 (자연스러운 전환)
- UI 의 `FlowEditor.tsx` / `BranchesFlowView.tsx` — 분기 트리거 편집 및 미리보기 연동 확인

### 완료 기준
- main 봇 → sub 봇 전환 시 컨텍스트 유실 없이 대화 이어짐
- sub 봇에서 다시 main 으로 복귀 가능 (또는 종료)
- `callbot-agents/[id]` 페이지에서 플로우 구성 반영

## 3. 완료 기준별 잔여 작업

### 3.1 컨텍스트(대화 이력 + 변수) 유실 없이 sub 봇 인계
- `_execute_tool_builtin` 의 runtime 재구성이 `state.history`, `state.var_ctx` 를 유지하는지 **직접 검증 필요**
- 자동 테스트: `backend/tests/` 에 main → transfer → sub 시나리오 — 직전 turn 에서 추출한 변수를 sub 봇에서 참조 가능한지

### 3.2 `branch_trigger` 평가 로직
**현재**: `branch_trigger` 는 LLM 이 `transfer_to_agent` 툴을 보고 인수로 직접 사용 — LLM 판단 단일 방식. 키워드 매칭 분기 평가 코드는 없음.

**티켓 본문**: "LLM 판단 or 키워드 매칭" 둘 다 옵션 제시.

**권고 (§5 결정 항목 #1)**: 현 코드 그대로 LLM 판단 단일 방식. 키워드 매칭은 별도 티켓으로 분리.
- 이유 1: LLM 이 툴 시그니처 + branch_trigger 텍스트를 보고 호출하는 동작이 이미 검증 가능
- 이유 2: 키워드 매칭은 sub-bot 트리 규모가 커지면 다시 필요해질 수 있는 미래 요구. 지금 추가 시 두 경로의 우선순위 정책이 복잡해짐
- 이유 3: 1인 운영 — 단순 경로 유지 ([[project_solo_operation]])

### 3.3 인계 시 greeting 없이 자연스러운 전환
**현재**: `_execute_tool_builtin` 의 인계 처리에 "안내 메시지 TTS" 가 포함됨.

**필드 추가**: `CallbotMembership.silent_transfer: bool` (콘솔 토글). 기본값 결정 필요 (§5 #2).
- true: 안내 멘트 없이 바로 sub 봇 첫 응답
- false: 짧은 안내 멘트 ("연결해 드릴게요") 후 sub 봇 응답

**권고**: 기본 false (안내 유지) — 사용자가 봇 전환을 인지하지 못하면 혼란.

### 3.4 sub 봇 → main 복귀 또는 종료
- `transfer_to_agent` 는 임의 봇 ID 수용 → main 봇 ID 로 호출 가능
- 검증: main 의 멤버십에 main 자기자신을 `branch_trigger="문의가 끝났을 때 메인으로 복귀"` 같은 형태로 노출하는지
- 또는 main 의 `CallbotAgent.global_rules` 에 종료 안내를 두고 별도 `end_call` 툴 사용 (현 코드 확인 필요)

### 3.5 `callbot-agents/[id]` 플로우 구성 반영
- 페이지에서 Save 누른 변경이 통화에 즉시 반영되는지 vs 다음 통화부터 적용인지 확인
- AICC-911 의 봇 버전 적용 시점 결정과 동일 정책 — **다음 통화부터** ([[project_callbot_agent_model]] 의 2026-05-11 결정과 정합)
