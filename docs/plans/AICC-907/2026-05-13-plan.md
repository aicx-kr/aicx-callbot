## 1. 티켓 원문

### 배경/목적
현재 `database_url = "sqlite:///./callbot.db"` 로컬 파일 DB + .env 직접 관리 상태로, 운영 환경에서 동시 통화 처리 및 데이터 영속성을 위해 PostgreSQL 과 Docker 기반 배포 환경이 필요하다.

### 작업 내용
- PostgreSQL 전환: `db.py` connection pool 설정, Alembic 마이그레이션 초기 설정
- Docker Compose: backend + postgres + frontend 컨테이너 구성
- 환경 분리: dev / staging / prod .env 체계 정립
- 헬스체크 엔드포인트 (GET /health) — DB 연결 상태 포함

### 완료 기준
- `docker compose up` 으로 전체 스택 로컬 실행 가능
- PostgreSQL 교체 후 기존 API 전체 테스트 통과
- `/health` → `{"status": "ok", "db": "connected"}` 반환

## 4. 본 문서가 추가하는 결정

### 4.1 LangSmith 환경변수 완전 배제
chatbot-v2 패턴을 참고하되 LangSmith 만은 제외 ([[project_callbot_no_langsmith]]).
- `.env.example`, `backend/pyproject.toml`, IaC (EKS manifest), SSM 어디에도 `LANGSMITH_*`, `LANGCHAIN_TRACING_*` 키 X
- 의존성에서도 `langsmith` 패키지 제거

### 4.2 헬스체크 변경
`app.py:110-114` 의 `/api/health` 의도(코드 작성 X, 형태만):
```
{
  "status": "ok",
  "db": "connected" | "degraded",
  "voice_mode_available": bool
}
```
- `db` 항목: `SELECT 1` 짧은 ping (300ms 타임아웃)
- 200 유지 — k8s livenessProbe 가 잘못 재시작 막기 위함
- 별도 `/readyz` 엔드포인트가 503 (DB 끊김 시 트래픽 차단용)

### 4.3 tenant-specific 환경변수 금지
페르소나·말투·프롬프트 등 테넌트별 설정은 .env 에 두지 않는다 ([[feedback_no_hardcoded_tenant_config]]). 본 PR 의 env 설계 시 `DEFAULT_PROMPT`, `DEFAULT_GREETING` 같은 변수 추가 금지. 그런 값은 DB + 어드민 콘솔에서만.

### 4.4 마이그레이션 컷오버 절차 (기존 §m 정합화)
1. Alembic baseline 생성 (현 SQLite 스키마 그대로) → autogenerate 검증
2. Postgres dev 인스턴스 (회사 dev common RDS 의 callbot DB) 에 baseline 적용
3. SQLite → Postgres 데이터 이관 스크립트 작성 — `backend/callbot.db`, `backend/callbot-share.db` 의 row 그대로 INSERT
4. 코드 PR: `db.py` 의 engine URL 을 env 변수 기반으로 + pool 옵션 추가
5. 회귀 테스트: `backend/tests/` 전체 + 수동 통화 1건
6. `_migrate_sqlite_add_columns()` 삭제 (Alembic 이 대체)
